## 2. 경기 시뮬레이션 화면 (P1 버그 수정 확인)

**수정 내용:** 빌드 선택 후 `BUILD_REVEAL_DELAY` 구간에서 전략 버튼 재클릭 시  
세트가 2회 처리되는 버그 수정 (`_set_running` 플래그, 커밋 `909f44b`)

### 확인 절차

1. 게임 시작 → 경기 준비 화면 → 경기 시작
2. 전략(초반집중 / 균형 / 후반체력전) 선택
3. 전술(공세 / 수비 / 기동) 선택
4. **전술 선택 직후 0.5~1초 이내에 전략 버튼을 반복 클릭**
5. **빌드 버튼이 다시 활성화되지 않아야 한다** ✓
6. 세트 결과가 정확히 1회만 반영되는지 스코어 확인 ✓
7. 다음 세트 진입 후 전략/전술 버튼이 정상적으로 초기화되는지 확인 ✓

### 합격 기준

| 확인 항목 | 기대 결과 |
|---|---|
| 전술 선택 후 전략 버튼 클릭 | 아무 반응 없음 |
| 스코어 (예: 1-0) | 정확히 1회만 올라감 |
| 다음 세트 전략 버튼 | 다시 활성화되어 선택 가능 |

## 7. 보안 규칙 (API 키 · 시크릿)

- API 키는 **절대 소스코드에 직접 작성하지 않는다**
- 환경 변수는 `.env` 파일에 보관한다 (`.gitignore`에 `.env*` 등록됨 — 자동 제외)
- `.env.example` 파일에 키 형식만 기록하고 실제 값은 비워둔다
- 커밋 전 `git diff` 로 민감 정보 포함 여부를 확인한다

### .env 파일 사용 방법

```bash
# 1. 템플릿 복사
cp .env.example .env

# 2. .env 에 실제 키 입력 (이 파일은 git 에 올라가지 않음)

# 3. Python 에서 읽기
import os
api_key = os.getenv("ANTHROPIC_API_KEY", "")
```
